# Approved Sources for Career Page Generation

Used by `generate_next_steps.py` to guide source selection in the prompt.

URL format: homepage URL only. The model must use the exact publication URL listed here as the `url` field in `sources[]`. Do not invent article-level paths — use web search to find the specific report page, then verify it in a browser before adding it to a card.

## Practitioner Voices (for howToAdapt quotes)

Prefer these for howToAdapt quotes over data-only sources. Named professionals discussing how their roles are changing.

**Priority: real named practitioners over report summaries.** Search for quotes from hiring managers, engineering leads, CTOs, or company executives describing a specific skill shift or workflow change. A quote attributed to "Sarah Vessels, GitHub Staff Engineer" or "Thomas Dohmke, GitHub CEO" is far stronger than one attributed to a report. Always search for the specific article URL and attribute to the named person + title.

## Avoid (all domains)

- Gartner, IDC, Forrester, MarketsandMarkets — paywalled, inflated projections, URL rot
- Forbes contributor network, The New Stack — low editorial bar
- Vendor blogs (e.g. Salesforce blog ≠ Salesforce State of Sales report)
- Career advice sites: Indeed editorial, LinkedIn Learning blog, Glassdoor editorial
- Opinion pieces without data
- Undated content
- Sources older than 2 years unless no newer equivalent exists
- Generic homepages as quote sourceUrls — must link to the specific report or article page
